# KitchenMate - 会员制临时工雇佣系统

基于TypeScript原生小程序开发的会员制临时工雇佣系统前端应用。

## 🎯 项目完成总结

本项目已完整实现基于原生微信小程序的会员制临时工雇佣系统，严格按照业务逻辑要求开发，具备以下核心特性：

### ✅ 已完成功能
- **完整的会员系统**: 支持会员注册、余额管理、角色切换（雇主/雇员）

### 🔧 技术实现亮点

- **TypeScript + WXML + WXSS**: 基于TypeScript的小程序技术栈
- **小程序组件系统**: 基于小程序原生组件开发模式

## 📋 项目概述

这是一个符合 iOS 设计风格的会员制临时工雇佣系统，实现了完整的业务逻辑：

- **会员管理**: 会员注册、充值、角色切换
- **任务管理**: 发布任务、接取任务、完成任务
- **资金闭环**: 雇主支付300元，雇员获得600元，系统保证资金平衡
- **数据统计**: 收益统计、任务统计、数据导出

## 🛠 技术栈

- **框架**: 微信小程序原生框架
- **语言**: TypeScript + WXML + WXSS
- **API 风格**: 小程序原生API（wx.* 系列接口，TypeScript类型安全）
- **状态管理**: 小程序页面数据绑定和组件通信机制
- **类型定义**: @types/wechat-miniprogram 提供完整API类型支持
- **样式**: CSS3 + iOS 设计规范

## 📁 项目结构

```
KitchenMate/
├── components/          # 🔧 可复用组件库
│   ├── KMButton/        # iOS风格按钮组件（主色/次色/危险色）
│   ├── KMCard/          # 卡片容器组件（阴影+圆角）
│   ├── UIListItem/      # 列表项组件（图标+标题+副标题）
│   ├── StatCard/        # 统计卡片组件（数值+标签）
│   └── TaskCard/        # 任务信息卡片组件
├── pages/               # 📱 页面模块
│   ├── index/           # 首页（概览+快捷操作）
│   ├── member/          # 会员中心（余额+角色管理）
│   ├── tasks/           # 任务大厅（发布/接取任务）
│   ├── mytasks/         # 我的任务（任务管理）
│   ├── stats/           # 数据统计（收益分析）
│   └── task-detail/     # 任务详情（详细信息）
├── utils/               # 🔧 工具函数
│   ├── dateUtils.ts     # 日期格式化工具
│   ├── numberUtils.ts   # 数字格式化工具
│   └── storageUtils.ts  # 本地存储工具
├── app.ts               # 📱 小程序入口文件
├── tsconfig.json        # 📋 TypeScript配置文件
├── app.json             # 📋 小程序全局配置
├── app.wxss             # 🎨 全局样式
└── static/              # 📦 静态资源
    └── tabbar/          # 底部导航图标
```

## 🚀 快速开始

### 📋 环境要求
- **微信开发者工具**: 最新版本（支持TypeScript编译）
- **Node.js**: 14.0 或更高版本
- **@types/wechat-miniprogram**: 微信小程序API类型定义
- **操作系统**: Windows/macOS/Linux

### 🔧 开发步骤
1. **安装微信开发者工具**: 下载并安装最新版微信开发者工具
2. **导入项目**: 使用微信开发者工具打开项目目录
3. **配置TypeScript支持**: 在项目设置中启用"使用TypeScript"
4. **安装类型定义**: `npm install --save-dev @types/wechat-miniprogram`
5. **配置项目**: 设置小程序AppID和基本信息
6. **运行调试**: 选择运行到模拟器或真机调试
7. **开始开发**: 修改代码并实时预览效果

### 📦 构建发布
1. **配置 app.json**: 设置小程序基本信息
2. **上传代码**: 在微信开发者工具中上传代码
3. **提交审核**: 在微信公众平台提交审核
4. **发布上线**: 审核通过后发布小程序

## 📊 业务规则

### 🔄 核心业务逻辑
1. **任务撮合**: 每个任务需要2个雇主和1个雇员参与
2. **资金流转**: 雇主支付300元，雇员获得600元收入
3. **角色切换**: 会员可以自由切换雇主/雇员角色
4. **资金安全**: 系统保证资金闭环，确保净收益≥0

### 📈 状态流转
- **任务状态**: 待接取 → 进行中 → 已完成
- **会员状态**: 雇主/雇员角色自由切换
- **资金状态**: 充值 → 支付 → 收入 → 统计
- **数据流向**: 实时同步，确保数据一致性
